## 基本语法

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### 段落和换行

- 段落之间空一行
- 行尾加两个空格实现换行

### 强调

```markdown
*斜体* 或 _斜体_
**粗体** 或 __粗体__
***粗斜体*** 或 ___粗斜体___
~~删除线~~
```

### 列表

无序列表：

```markdown
- 项目1
- 项目2
  - 子项目1
  - 子项目2
```

有序列表：

```markdown
1. 第一项
2. 第二项
   1. 子项1
   2. 子项2
```

### 链接和图片

链接：

```markdown
[显示文本](URL)
```

图片：

```markdown
![替代文本](图片URL)
```

### 代码

行内代码：

```markdown
`print("Hello World")`
```

代码块：

````markdown
```python
def hello():
    print("Hello World")
```
````

### 引用

```markdown
> 这是引用内容
> 可以多行
```

### 分隔线

```markdown
---
或
***
或
___
```

### 表格

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
|:-------|:-------:|-------:|
| 数据1  |  数据2  |  数据3 |
| 数据4  |  数据5  |  数据6 |
```

## 最佳实践

1. 标题前后空一行
2. 列表项前后空一行
3. 代码块使用特定语言标记
4. 长表格考虑使用HTML表格
5. 复杂布局考虑使用HTML+CSS

记住，Markdown的目标是易读易写，保持简洁明了最重要。
